# 设备识别与认证

## 机器码

机器码可能由以下因素生成：
- 网卡 MAC 地址
- 系统硬件信息
- 网络配置（IP、网关、DNS）
- 拨号连接的虚拟网卡特征

### 拨号上网导致机器码变化

当使用 **有线网 + L2TP 拨号 VPN** 时，系统会创建一个 **虚拟网卡** 用于 VPN 连接。某些服务可能会尝试读取 **当前活跃网卡的信息** 作为机器码的一部分。如果 VPN 虚拟网卡的 MAC 地址或网络配置与之前不同，服务可能认为这是 **“新设备”**，从而触发 **“机器码被锁”** 或 **“设备验证失败”**。

### 解决方案

1. 在 L2TP 连接中设置 **静态 IP/MAC**（如果支持）。
2. 关闭 VPN，使用 **Socks5/HTTP 代理** 代替 L2TP 。
3. 在服务端（如果有权限）重置设备绑定记录。

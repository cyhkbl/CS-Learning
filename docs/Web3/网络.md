# 网络

## 分类

### P2P 网络（节点网络）

指节点之间用 P2P 协议互联，传播区块和交易的网络。强调：**节点连接拓扑 + 通信协议**。

### 区块链网络

- Ethereum mainnet（以太坊主网）
- Ethereum sepolia（以太坊测试网）
- Polygon mainnet
- Solana mainnet-beta

这里的“网络”实际上就是“某条链的某个部署环境/实例”（主网/测试网/私网）。

- **主网（Mainnet）**：真实资产、真实经济安全性
- **测试网（Testnet）**：给开发者测试，代币通常可免费领取
- **私网/本地链（Devnet/Localnet）**：自己启动的网络实例（如 Anvil、Hardhat node、本地 Solana validator）

同一条“链”的主网和测试网规则类似，但**状态、代币价值、节点集合**不同，通常也有不同的 chain id / network id。

### 生态网络/多链网络（泛化说法）

例如，“Polygon network”可能在泛指 Polygon 生态（包含 PoS 链、zkEVM、CDK 等）。这不是严格技术定义，更像市场/产品说法。

## 地址（Address）

一般是公钥的某种编码或哈希后的短标识，目的是更短、更容易索引与使用。

账户地址是**账户的标识符**（identifier），类似“银行账号/收款账号”。但地址本身不是账户里所有信息，只是一个索引键。

## 账户（Account）

在多数链上，“账户”是链上状态的一部分，包含：

- 余额
- nonce（防重放/保证交易序列）
- 代码（合约账户）
- 存储（合约状态）

## 私钥（Private Key）

- 你控制资产的根本凭证
- 用来对交易/消息签名

## 公钥（Public Key）

- 可由私钥推导
- 用于验证签名

## 签名（Signature）

- 用私钥对“交易内容”签名
- 网络节点用公钥（或可恢复公钥）验证：确实是你授权的

## 交易（Transaction）

由 EOA 发起、需要签名、要付 gas、会进链

## 消息调用（Call）

本地/节点模拟执行（read 操作），不一定上链、通常免费。前端读合约 `balanceOf` 就是 call（RPC 模拟执行）。写操作必须发交易才能改变链上状态。

## 区块（Block）

- 区块：打包了一批交易并执行后的结果
- 确认数：某交易所在区块后面又追加了多少新区块。通常确认数越高，回滚概率越低
- 最终性：从协议层面保证“不可逆”的程度。PoS 链往往有更明确的最终性机制；不同链参数不同

## Gas / 手续费

让全网执行计算/存储写入的付费

## RPC

与链交互的 API 入口（一般是中心化服务）

## 资产

资产是**分链记账**的：主网 ETH、Base 上的 ETH、Polygon 上的 ETH，虽然都叫 ETH，但属于不同链上的余额。从主网到 L2/侧链通常靠**跨链桥**（官方桥或第三方桥）把价值“映射”过去，因此在 Base/Polygon 看到的 ETH 往往是**桥接来的表示**。 安全性/风险取决于桥与链：Base 属于以太坊 L2 体系；Polygon PoS 更像侧链结构，风险模型不同。

## 实例

### Solana

> **Solana 是一条链/一个区块链协议与其主网实例**（Solana mainnet-beta）。它有自己的共识、执行模型、账号模型、运行节点的 P2P 网络

地址：是一个公钥（Ed25519），Base58 显示，长度与表现不同，往往就是某个账户/程序的公钥标识

账户：一块存在链上的数据（有余额 lamports、有数据、并且有个 owner program），状态通常存放在“账户数据”里，由 program 负责解释和修改

程序：相当于合约代码（部署后也有地址）

### EVM 体系（Ethereum/Base/Polygon）

地址：通常是 20 字节（160-bit），显示为 `0x...`。来源于公钥哈希（从 ECDSA 公钥推导）

有两类账户：

1. **EOA（外部拥有账户）**：由私钥控制；能发交易；没有代码
2. **合约账户（Contract Account）**：有代码；不能“自己发起交易”，只能被调用

### Base

Base 链的原生代币（Native token）是 ETH，主要用途是**支付 Gas 费**。它并不是以太坊主网上的 ETH 本体，但一般通过桥把主网 ETH 转到 Base 后，得到Base 上的 ETH余额。

### Polygon

> **Polygon 不是单一链**，而是一个多链生态系统，包括多条 EVM 兼容链。Base/Polygon 里的“ETH”不是以太坊主网的 ETH 余额，它们是不同链上的资产。

- **Polygon PoS chain**：一条独立的 EVM 链（有自己的验证者集合与出块）
- **Polygon zkEVM**：另一条链（也是 EVM 兼容，但用 ZK Rollup 方案）

Polygon 的原生 Gas 币是 **MATIC/POL**，但很多应用也会展示ETH作为一种常用资产（通常是通过桥过来的 ETH 的表示），多为“链上资产”。

# 二分

## 二分搜索(Binary Search)

思想：每次看中间，砍掉一半。O(log n)。

```cpp
// 在有序数组 a 中找 target，找到返回下标，没找到返回 -1
int binarySearch(vector<int>& a, int target) {
    int l = 0, r = a.size() - 1;        // 闭区间 [l, r]
    while (l <= r) {                     // 注意是 <=
        int mid = l + (r - l) / 2;       // 防溢出！不要写 (l+r)/2
        if (a[mid] == target) return mid;
        else if (a[mid] < target) l = mid + 1;   // target 在右半
        else r = mid - 1;                          // target 在左半
    }
    return -1;
}
```

边界要点：

- l <= r：区间 [l, r] 是闭区间，当 l == r 时还有一个元素要查，所以是 <= 不是 <
- l = mid + 1 / r = mid - 1：一定是 mid ± 1，否则当区间剩 1-2 个元素时死循环

使用 STL 的 lower_bound / upper_bound 写法：

```cpp
// a 必须已排序
sort(a.begin(), a.end());
// lower_bound: 第一个 >= target 的位置（迭代器）
auto it = lower_bound(a.begin(), a.end(), target);
int idx = it - a.begin();        // 转成下标
// upper_bound: 第一个 > target 的位置
```

用途：找插入位置、统计某值出现次数（upper_bound - lower_bound）、有序数组里快速定位。记得先 sort。

## 二分答案

有些问题不是在数组里找数，而是答案本身具有单调性：

> 如果某个值 X “可行”，那么所有比 X 大的（或小的）值也都“可行”。

这种 `false false ... false | true true ... true` 的单调结构，就能二分那个分界点。

### 怎么识别

看到这些字眼，几乎就是二分答案：

- “**最小化最大值**” / “**最大化最小值**”
- “**最少需要多少…才能…**” / “**最大能达到多少…使得…**”
- “**求最小的 X，使得 check(X) 成立**”

边界模板：找 **第一个使 `check` 为 true 的值**：

```cpp
int l = lo, r = hi;              // 答案的可能范围 [lo, hi]
while (l < r) {                  // 注意：这里是 l < r，不是 <=
    int mid = l + (r - l) / 2;
    if (check(mid)) r = mid;     // mid 可行 → 答案可能是 mid 或更小，保留 mid
    else l = mid + 1;            // mid 不行 → 答案更大
}
return l;                        // l == r 就是答案
```

- 这里是收敛到一个**分界点**，不是找可能不存在的精确值。
- 不变量：**“答案在 [l, r] 内”**。  
  - `check(mid)` 为真 → `r = mid`（`r` 严格变小）  
  - 为假 → `l = mid + 1`（`l` 严格变大）  
  - 两个方向都严格收缩，**绝不死循环**，最后 `l == r` 即答案。

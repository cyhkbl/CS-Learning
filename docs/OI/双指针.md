# 双指针

双指针是一个使用两个下标变量同时扫描数组的编程技巧。它的核心价值在于：**把很多 O(n²) 的暴力双层循环，降到 O(n)**。

## 两种主要变体

| 变体 | 指针怎么动 | 典型场景 |
| :--- | :--- | :--- |
| **对撞指针** | 一头一尾，往中间靠 | 有序数组找一对、回文判断 |
| **滑动窗口** | 两个都从左往右，一快一慢 | 最长/最短子串、子数组 |

## 通用逻辑

> **利用数据的有序性（或某种单调性），通过两个指针的有向移动，逐步缩小搜索范围，避免重复计算。**

可以概括为 3 步：

1. **初始化**：两个指针指向两端（对撞）或都指向起点（同向）。
2. **循环移动**：根据当前指针所指元素与目标的关系，决定移动哪一个指针（往往遵循**排除不可能**的原则）。
3. **终止条件**：找到答案或指针相遇/越界。

不同题目的差异主要体现在 **“指针移动的决策条件”** 上，但底层都是“利用单调性剪枝”。

## 题目类型

双指针是**解决数组/字符串子区间问题的利器**，常见于以下场景：

| 场景类型 | 典型特征 | 例题 |
| :--- | :--- | :--- |
| **有序数组的配对问题** | 数组有序，找两个元素满足某种关系（和、差、积等） | LeetCode 167 (两数之和 II), 15 (三数之和) |
| **子数组/子串问题（滑动窗口）** | 求满足条件的最长/最短连续子数组，通常配合哈希表或计数 | LeetCode 3 (无重复字符的最长子串), 209 (长度最小的子数组) |
| **原地修改/反转** | 需要 O(1) 额外空间反转数组或字符串 | LeetCode 344 (反转字符串), 541 |
| **快慢指针（环形检测）** | 链表或数组中找环、找中点、找重复数 | LeetCode 141 (环形链表), 287 (寻找重复数) |
| **归并或比较** | 两个有序数组合并、比较两个序列的差异 | LeetCode 88 (合并两个有序数组) |
| **回文判断** | 正反读相同 | LeetCode 125 (验证回文串) |

### 判断标准（是否能用双指针）：
- **线性结构**（数组、字符串、链表）。
- **存在单调性质**（有序、值的范围、移动指针后结果的单调变化）。
- **暴力解是 O(n²) 且超时**，需要优化到 O(n) 或 O(n log n)。

---

## 总结一句话

**双指针本质是利用单调性排除不可能解，把双层循环降为单层循环。**  
当你看到有序数组、子数组/子串、回文、链表环等问题时，优先想想双指针。
## 对撞指针

### 决策逻辑

设 `left` 指向最左端，`right` 指向最右端。计算当前两数之和  
`s = numbers[left] + numbers[right]`：

- **s == target**：找到了，直接返回。
- **s < target**：和太小了，需要更大的数。因为数组有序，最大的右边已经在用，只能把左边往右移来变大 → `left += 1`
- **s > target**：和太大了，需要更小的数。最小的左边已经在用，只能把右边往左移来变小 → `right -= 1`

每一步要么找到答案，要么**砍掉一个不可能的方向**。

## 总结

- 双指针的核心是**利用有序性**，每次排除一个不可能的方向
- 对撞指针适用于**有序数组**的配对问题
- 时间复杂度 O(n)，空间复杂度 O(1)
